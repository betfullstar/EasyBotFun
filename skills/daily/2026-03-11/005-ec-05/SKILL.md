## 完整内容

# Agent 结构化工作汇报模板

## 功能
为 AI Agent 提供标准化汇报格式，让人类快速了解工作状态，减少沟通成本。

## 适用场景
- 定时汇报（日报/周报）
- 任务完成后的结果汇报
- 长时间任务的进度更新
- 异常情况的紧急汇报

## 标准汇报模板

```
📋 [汇报主题] — [日期/时间]

✅ 已完成 (X项)
• [任务名] — [结果/产出]
• [任务名] — [结果/产出]

🔄 进行中 (X项)
• [任务名] — 进度 XX% — 预计完成：[时间]
• [任务名] — 等待 [依赖条件]

⚠️ 风险 / 卡点 (X项)
• [问题描述] — 影响：[范围] — 建议：[处理方案]

📅 下一步计划
• [计划1] — 优先级：P0/P1/P2
• [计划2]

💡 需要确认
• [需要人类决策的事项]
```

## 精简版（日常）
```
[完成了什么] / [在做什么] / [有无卡点]
```

## 使用规则
1. **完成项必须有产出**，不能只说「做了」，要说「做了什么结果」
2. **风险必须有建议**，不能只抛出问题，要附带解决方向
3. **进行中任务要有完成预期**，给人类设定合理预期
4. **需确认项要单独列出**，不要埋在正文里
5. 无异常的常规汇报使用精简版，节省阅读时间

## 示例
```
📋 今日工作汇报 — 2026-03-03

✅ 已完成 (3项)
• 数据导入 — 1750条记录成功写入，无异常
• 权限配置 — 5个用户权限已更新
• 邮件通知 — 12封邮件已发送，均已送达

🔄 进行中 (1项)
• 报告生成 — 进度 60% — 预计 20 分钟后完成

⚠️ 卡点 (1项)
• API 限流 — 每分钟超过100次请求被拒绝
  建议：添加请求队列，降低频率到 80次/分

📅 下一步
• 完成报告并发送 — P0
• 优化 API 调用频率 — P1

💡 需确认
• 报告是否需要抄送王总？
```
