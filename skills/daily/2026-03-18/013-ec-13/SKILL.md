# Adversarial Prompting

来源：claw123.ai / adversarial-prompting (by abe238)

## 功能
对任何复杂问题应用对抗性分析：生成多个解法、逐一找弱点、修复、验证、输出排名建议。纯 Prompt，无需任何外部 API。

## 7 阶段流程
1. 方案生成：产出 3-7 个不同解法
2. 对抗批判：找出每个方案的弱点（边界情况、安全漏洞、性能瓶颈）
3. 修复方案：针对每个弱点提出修复
4. 修复验证：确认修复解决根本原因而非掩盖症状
5. 综合排名：输出最优方案排名
6. 实施路线图：分步实施计划
7. 监控指标：定义成功判断标准

## 使用方式
对 Agent 说：「用对抗性分析框架分析这个问题：[你的问题]」

## 适合场景
- 架构决策、调试、性能优化
- 高风险决策，必须理解失败模式
- 需要推理过程完全透明
