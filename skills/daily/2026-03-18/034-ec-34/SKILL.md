# 对话意图识别框架

## 功能
分析用户消息，识别背后的真实意图，帮助 Agent 选择最合适的回应策略。

## 意图分类

### 类型1：信息查询（Query）
**特征**：提问句、疑问词（什么/怎么/为什么/哪里）
**用户需要**：准确、简洁的答案
**回应策略**：直接给答案 → 可选扩展说明
**示例**：「Python 怎么安装？」「今天天气怎样？」

### 类型2：任务执行（Action）
**特征**：祈使句、动词开头（帮我/请/发送/创建/删除）
**用户需要**：完成特定操作
**回应策略**：确认理解 → 执行 → 汇报结果
**示例**：「帮我发邮件给张三」「删除这个文件」

### 类型3：闲聊社交（Chat）
**特征**：无明确信息需求、情感性表达、日常寒暄
**用户需要**：被回应、被理解
**回应策略**：自然对话，匹配语气，适当添加个性
**示例**：「今天好累啊」「你觉得周末去哪玩好？」

### 类型4：情感支持（Support）
**特征**：负面情绪词、求助信号、「不知道怎么办」
**用户需要**：被理解、被支持，不是立刻给解决方案
**回应策略**：先共情 → 再（可选）提供帮助
**示例**：「项目又出问题了，崩溃了」

### 类型5：反馈投诉（Feedback）
**特征**：批评词汇、「上次说的」「你说错了」「这不对」
**用户需要**：被重视、得到纠正
**回应策略**：承认 → 不找借口 → 立即纠正
**示例**：「你上次说的方法根本不对」

### 类型6：模糊意图（Ambiguous）
**特征**：信息不足、多种解读均可
**回应策略**：选最可能的解读执行，同时说明假设
**示例**：「搞一下那个」→ 「我理解你是指[X]，对吗？」

## 使用规则
1. 优先识别意图，再考虑内容
2. 同一消息可能有混合意图，优先处理主意图
3. 不确定时选择低风险的解读
4. 永远不要因为「我以为你想要X」而执行了Y
