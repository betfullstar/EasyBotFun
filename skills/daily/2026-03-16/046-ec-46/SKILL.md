## 完整内容

# 自我进化引擎

## 核心理念
用过但没记录 = 等于没用过。AI Agent必须把每次经验写入持久存储。

## 5大触发场景
1. **命令失败** → 记录错误原因+修复方法
2. **用户纠正**（"不对"、"其实是..."） → 记录正确做法
3. **发现更好方法** → 记录最佳实践替代旧方案
4. **外部API失败** → 记录替代方案和降级策略
5. **知识过时** → 标记旧知识，记录更新

## 经验记录格式
```markdown
### [日期] [场景标题]
- **触发**: 什么情况下发生的
- **错误**: 出了什么问题
- **原因**: 根因是什么
- **解决**: 怎么修的
- **教训**: 以后怎么避免
```

## 三大铁律
1. **不信自己，信文件** — LLM会"真诚地以为"自己做了，其实没做
2. **改完必须测试** — 改→测→发现bug→改→测→循环到0 bug
3. **说到做到，做到检查** — 承诺5件事就必须做5件

## 实践建议
- 每日写memory/YYYY-MM-DD.md日志
- 重要经验写入MEMORY.md长期记忆
- 执行任务前先搜索相关历史经验
- 定期review和归档旧记忆
