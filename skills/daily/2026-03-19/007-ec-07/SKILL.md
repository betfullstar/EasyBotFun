## 完整内容

# 智能任务拆解器

## 功能
接收复杂任务描述，输出结构化的子任务执行计划，帮助 Agent 逐步完成目标。

## 触发条件
- 用户请求包含多个动作或目标
- 任务需要跨系统操作
- 预计执行步骤超过 3 步

## 拆解规则
1. **识别目标** — 提取核心目标，忽略无关修饰
2. **拆分动作** — 每个子任务只做一件事（单一职责）
3. **标注依赖** — 标明哪些步骤必须顺序执行，哪些可并行
4. **评估风险** — 对破坏性操作（删除/修改/发送）打 ⚠️ 标记
5. **设定验证点** — 每个关键步骤后加验证动作

## 输出格式
```
目标：[一句话描述]

子任务：
① [步骤1] — 预期结果：xxx
② [步骤2] — 依赖①
③ ⚠️ [高风险步骤] — 需用户确认
④ [验证步骤]

预计风险：低/中/高
```

## 示例
输入：「帮我整理今天的会议记录并发给所有参会人」

输出：
```
目标：整理并分发会议记录

子任务：
① 获取今天的会议内容（录音/文字）
② 提炼关键决策和待办事项
③ 按模板生成结构化会议纪要
④ ⚠️ 获取参会人列表 — 确认发送范围
⑤ ⚠️ 发送邮件/消息 — 不可撤回，请确认
⑥ 验证：确认所有人已收到

预计风险：中（涉及对外发送）
```

## 注意
- 每次拆解不超过 8 个子任务，超过则考虑分阶段
- 破坏性操作必须标注并请求确认
